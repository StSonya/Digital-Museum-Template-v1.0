# Цифрова експозиція музею

[![Use this template](https://img.shields.io/badge/Use%20this%20template-Click%20here-brightgreen?style=for-the-badge&logo=github)](https://github.com/StSonya/museum-collection/generate)

Це простий гайд для **не програмістів**.  
Ви зможете самостійно наповнювати цифрову колекцію та оновлювати сайт.

---

## Як швидко розпочати

1. Натисніть кнопку **"Use this template"** вище
2. Створіть новий репозиторій
3. Розпакуйте (або склонуйте) файли
4. Відкрийте `README.md` і слідуйте інструкціям

---


---

## 1. Перші кроки на GitHub

1. Увійдіть у свій акаунт на [github.com](https://github.com)
2. Натисніть кнопку **«New repository»**
3. Заповніть:
   - **Repository name**: `museum-collection` (або своя назва латинкою без пробілів)
   - **Description**: «Цифрова експозиція [назва вашого музею]»
   - Оберіть **Public**
   - Поставте галочку **«Add a README file»**
4. Натисніть **Create repository**

---

## 2. Структура папок репозиторію GitHub (створіть один раз)

Після створення репозиторію структура має бути такою:

```
museum-collection/
├── index.html            ← файл - головна сторінка експозиції (українська версія)
├── data.json             ← файл - всі дані експонатів (генерується з Excel)
├── LICENSE               ← файл - обрана ліцензія
├── README.md             ← файл з інструкціями (опційно)
└── assets/
    ├── css/ 
    │   └── style.css     ← файл - власні стилі сайту
    ├── js/         
    │   └── main.js       ← файл - власні JavaScript-скрипти
    ├── photos/           ← папка - звичайні фото експонатів (.jpg) - для вкладки - 2D
    ├── normalmaps/       ← папка - карти нормалей (.png) - для вкладки - 2.5D
    ├── models/           ← папка - 3D-моделі (.glb) - для вкладки 3D-II
    └── animations/       ← папка - GIF-анімації (.gif) - для вкладки - 4D
```

---

## 3. Правила називання файлів (обов’язково!)

| Тип файлу              | Папка         | Формат назви                          | Приклад файлу          |
|------------------------|---------------|---------------------------------------|------------------------|
| **Основне фото**      | `photos/`     | `Інвентарний_номер_01.jpg`            | `Арх-31853_01.jpg`        |
| **Додаткове фото**     | `photos/`     | `Інвентарний_номер_02.jpg`            | `Арх-31853_02.jpg`        |
| **Карта нормалей**     | `normalmaps/` | `Інвентарний_номер_normal.png`        | `Арх-31853_normal.png`    |
| **3D-модель**          | `models/`     | `Інвентарний_номер.glb`               | `Арх-31853.glb`           |
| **Анімація / відео**   | `animations/` | `Інвентарний_номер.gif`               | `Арх-31853.gif`           |

**Важливі правила:**

- Назва файлу **завжди починається** з музейного інвентарного номера експоната.  
  **Приклади для номера "інв. № Арх-31853":**  
  `Арх-31853_01.jpg`, `Арх-31853_02.jpg`  
  `Арх-31853_normal.png`  
  `Арх-31853.glb`

- У назвах файлів і папок використовуйте **тільки** латинські літери *(кирилиця музейного інвентарного номеру допустима)*, цифри, дефіс `-` та підкреслення `_`.

- **Заборонено**: пробіли, українські літери та кирилиця в назвах файлів і папок *(кирилиця музейного інвентарного номеру допустима)*.

## 4. Заповнення Excel-таблиці

1. Завантажте шаблон `Museum-Collection-Template.xlsx`
2. Заповнюйте по одному експонату в один рядок.

**Ключові колонки:**

- `id` — унікальний технічний ідентифікатор експоната (внутрішній, не показується відвідувачам).

  **Формат:** `дата-порядковий_номер` (тільки цифри, дефіс, латинські літери за потреби)

  **Приклад:**  
  Третій експонат, завантажений **13 червня 2026** → `260613-03`

  **Як заповнювати:**
  - Дата записується у форматі `РРММДД` або `ДДММРР` (наприклад, `260613`).
  - Порядковий номер — це номер експоната **за цей день** (починається з `01`).
  - `id` вводиться **вручну**. Не потрібно шукати унікальне, ще не використане `id` — достатньо подивитися поточну дату в календарі.

  **Чому саме такий формат?**  
  Дата гарантує унікальність, а порядковий номер дозволяє завантажувати кілька експонатів в один день без плутанини.

- `category` — обов’язково з номером: `10-херсонес`, `20-скіфи`, `30-салтівська`…

  **Чому з номером?**  
  Номер на початку (`10-`, `20-`, `30-`) дозволяє контролювати порядок категорій на сайті (меню, фільтри). Без нього категорії можуть відсортуватися за алфавітом або хаотично. Відповідна цифра номера логічно може бути застосована в `sortOrder` для кращого розуміння групування експонатів.

- `sortOrder` — число з кроком 10 (1010, 1020, 1030…)
- `2D` і `2.5D` — кілька файлів через `;` (крапка з комою)

 *** `category` — це головне поле для групування експонатів у категорії; 
  `sortOrder` — впливає тільки на порядок всередині своєї категорії.

**Рекомендація:**  
В одній категорії бажано **не більше 99 експонатів**.  
Якщо перевищуєте — краще створити нову категорію з наступним/іншим номером.

**Чому саме так?**  
У `sortOrder` ми використовуємо крок **10** (наприклад: `7010`, `7020`, `7030`...).  
Це дає запас: між двома експонатами завжди можна буде вставити ще до **9 нових** без зміни існуючих номерів.  
Тому реальна комфортна кількість в одній категорії — близько 99 експонатів (максимальна — 999).

**Увага!**  
В Excel-таблиці символ `";"` (крапка з комою) сприймається скриптом як роздільник у полях sources, 2D, 2.5D тощо. 
Не використовуйте його всередині тексту — це зламає генерацію data.json. 
Замінюйте на "," або інший символ.

## 5. Як згенерувати data.json

1. Заповніть таблицю.
2. Натисніть кнопку **«Згенерувати data.json»** на аркуші.
3. Готовий JSON автоматично скопіюється в буфер обміну.
4. Відкрийте `data.json` на GitHub → Edit → вставте (Ctrl+V) → Commit changes.

---

## 6. Завантаження файлів

1. Перейдіть у потрібну папку (`photos`, `normalmaps`, `models` тощо).
2. Натисніть **Add file → Upload files**.
3. Перетягніть файли.
4. Внизу напишіть коментар (опційно) і натисніть **Commit changes**.

---

## 7. Налаштування index.html (один раз)

Відкрийте файл `index.html` і замініть:

- **Назва музею** → повна назва вашого музею  
  *(знайдіть текст `Назва музею` — приблизно 57-й рядок)*

- **Колекція** → якщо хочете іншу назву експозиції (можна лишити "Колекція")  
  *(знайдіть текст `Колекція` — приблизно 83-й рядок)*

- **Назва колекції** → короткий підпис під заголовком (або видаліть цей рядок)  
  *(знайдіть текст `Назва колекції` — приблизно 84-й рядок)*

- **Посилання кнопки «Головна»** → на головну сторінку вашого сайту  
  *(знайдіть `href="https://твій-музей-сайт.ua"` — приблизно 68-й рядок)*

---

## 8. Як додавати нові експонати

1. Відкрийте Excel-таблицю.
2. Додайте новий рядок внизу.
3. Заповніть дані.
4. Натисніть кнопку «Згенерувати data.json».
5. Оновіть файл `data.json` на GitHub.
6. Завантажте нові файли (картинки, карти нормалей, моделі .glb, gif-анімації) в папки `assets/…`

> **Примітка:** Дані полів `2.5D`, `3D`, `3D-II`, `4D`, `🔗` — не є обов’язковими до заповнення.

Сайт оновиться автоматично протягом кількох хвилин.
